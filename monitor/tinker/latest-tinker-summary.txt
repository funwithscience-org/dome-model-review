THE BUG (confirmed empirically):
  prune-integrity.js uses fs.statSync(file).mtimeMs to decide which files
  are past retention_days. But the script runs from a fresh `git clone`
  every scheduled run, and `git checkout` sets mtime = clone-time on
  every checked-out file. So st.mtimeMs ≈ NOW for every file,
  retention_days never matures, and 1,149+ files accumulate forever.
