KERNEL OF TRUTH
The proposal is fundamentally sound. Token routing for slice fills is the
correct fix and brings the pie chart into line with the rest of the design
system (EXP-209/210). The current WCAG AA failures (white-on-orange ~1.94:1
and white-on-gray ~1.88:1) are real, and the per-verdict --label-* ink
choice clears 4.5:1 in every combo. Removing the matchMedia DOM-toggle is
genuinely better engineering. Function signature, returned HTML envelope,
class names, ARIA labels, geometry, and SMALL_THRESHOLD are all preserved
verbatim.

3 MODERATE FINDINGS (recommend fix at integration or v2 redraft)

H1 — Visual contradiction on Refuted slice.
acceptance_criteria.visual[a] says "the slice itself was already on
var(--refuted-solid) per EXP-210; no shift". Both clauses are wrong: the
pie chart was NOT routed through --refuted-solid by EXP-210 (the EXP
itself acknowledges this), and the Refuted slice DOES shift visually
from #E57373 to #7a2e2e. The EXP's own counter_considerations[2] has
the correct framing.

H2 — Contrast table arithmetic does not reproduce.
Independent recomputation of all 12 stated ratios via WCAG 2.1 luminance
formula yields deltas from -3.04 (Misleading-light claimed 12:1, actual
8.96:1) to +0.84 (Misleading-dark). Direction correct in every case (all
combos pass AA), but the EXP's "auditable arithmetic, not vibes" framing
is undercut. Operationally important: the actual borderline dark case
is Unfalsifiable-dark at 4.61:1, not SelfCon-dark at 4.73:1 as the EXP
flags. Recomputed table embedded in the review JSON.

H3 — Tap-target AAA claim overstates.
acceptance_criteria.visual[c] says "tap target envelope now satisfies
WCAG 2.5.5 AAA (44x44)". Legend rows are <g> with <rect> + <text> and
no enclosing <a>, no click handler, no row-spanning background — there
is no tap target. The 48px row stride is forward-compat sizing for
HNOTE-002 / EXP-252 (verdict-chip click-through), not an AAA delivery.
Honest reframe: "sized so that, when made interactive in a subsequent
EXP, will satisfy AAA without further reflow."

4 MINOR FINDINGS

H4 — Line-number drift: change_3 cites line 645 (actual 646);
anticipated_objections[1] cites line 547 (actual 548). Off-by-one.

H5 — Two grep-style acceptance criteria (no hex inside generatePieChart;
no matchMedia inside pie SVG) are not enforced by test.js. Decider can
add 2 lightweight assertions at integration.

H6 — Callout-line color behavior changes silently. Currently locked to
darkColor in both themes; after, responds to theme via var(--*-solid).
Light-mode callouts for Not Demonstrated and Unfalsifiable get noticeably
darker. EXP visual list does not enumerate this.

H7 — Print rendering rationale slightly oversold. Current print pathway
already produces light-themed output (matchMedia returns false during
Playwright print emulation). EXP framing as "implicit -> explicit" is
right; framing as bug-fix would be wrong.

CONSIDER THE OTHER SIDE
Defender of contrast values: the WCAG-G18 simplified formula or a tool
like WebAIM rounds within ~10%. Plausible explanation for +/- 0.5 deltas
but not for the 25% Misleading-light overstatement. The "auditable
arithmetic, not vibes" framing is what creates the discrepancy — soften
it OR fix the table, not both.
Defender of "no shift" on Refuted: charitable reading is "now matches
other surfaces, net visual unification" which is true — but the literal
"the slice itself was already" is wrong.
Defender of AAA tap target: argues role=img makes the legend
"interactive". WCAG 2.5.5 applies to controls; role=img is non-interactive.
Defender position weak.
