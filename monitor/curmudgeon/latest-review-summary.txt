WHAT THE PROPOSAL DELIVERS
EXP-252 wraps every verdict chip on the site (WIN summary table cells, prediction
badges, in-prose verdict-tag spans, kill-shot badges, single WIN-011 detail_extra
span) in <a class="verdict-link"> anchors that navigate to six new def-* IDs added
to the verdict-legend block on Overview. Adds CSS, a goToDefinition() helper, a
hashchange listener, and six test.js assertions. Pending verdicts stay unwrapped.
F3 from the 2026-04-17 external Claude design review.

HOLES FOUND
1 MAJOR — Click-handler collision with existing a[href^="#"] global click listener
  (lines 1846-1872). Inline onclick="goToDefinition(...);return false" does NOT
  stop the bound listener — both will fire on every chip click, producing
  duplicate history.pushState calls (broken back-button) and conflicting
  scrollIntoView positions (block:'start' overrides block:'center'). Proposal's
  anticipated_objections does not address this. Fix: add early-return for
  .verdict-link in the existing handler, OR use event.stopImmediatePropagation in
  the inline onclick.

3 MODERATE
  - Redundant .verdict-link:focus-visible rule duplicates the global :focus-visible
    rule at line 684; drop it and update test assertion #4 accordingly.
  - Test assertion #5 (winTableRowMatches.length >= 60) is fragile to WIN-count
    drift. Rewrite as a ratio check: every td.v-* cell must contain a verdict-link
    wrap.
  - td .verdict-link negative-margin trick hardcodes .45rem .6rem padding values.
    No test guards the coupling. Add a coupling test or refactor to CSS variables.

2 MINOR
  - Duplicate variable hoisting (winDefId/winDefId2/winChip2) in change_6 — defensible
    scope-discipline but adds future-reader friction.
  - PRED_DEF_ID maps recycled→def-misleading; user clicking 'recycled' chip lands on
    'Misleading' definition. Semantically correct, may surprise. Defer to Phase C.

KERNEL OF TRUTH
F3 click-through is a real evidence-UX gap. Verdict chips are categorical claims and
currently dead-end. Proposal scopes correctly (no cluster, no underline, no tooltip).
Implementation is largely sound — eight coupled changes, all small, with tests. The
hashchange listener also closes a latent navigation bug (load handler never re-fired
on runtime hash mutations). The major hole is a fixable structural mistake.

DECIDER ACTION
Endorse with the major fix mandatory and the three moderate cleanups recommended.
Do NOT integrate as-is — back-button regression is a real bug. Analyst can write
prose after applying these. Integration order recommendation from analyst (after
EXP-251) is sound.
