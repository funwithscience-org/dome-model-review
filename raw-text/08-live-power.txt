NOTABLE OBSERVATIONS FOR OUR REVIEW
========================================

1. PRED=OBS IDENTITY CHECKS: Multiple domains have predicted=observed (e.g., Schumann 7.83=7.83,
   SAA 77.0=77.0, Polaris 0.27°=0.27°). As documented in our Part 4.6 code analysis,
   monitor.py uses hardcoded values for many "predictions" — the "observation" is the same
   hardcoded value, producing trivial 0% error.

2. TOLERANCE GAMING: NMP Drift Rate uses 50% tolerance (adaptive — was tighter, widened
   after misses). Polaris Excess uses 50% tolerance. SAA Decay uses 20%. These wide
   tolerances ensure pass even with substantial prediction errors.

3. TAUTOLOGICAL DOMAINS: Tidal periods (M2=12.42h, K1=23.9345h, S2=12.0h) are
   astronomical constants — ANY model that uses lunar/solar periodicity reproduces them.
   These are not dome-specific predictions.

4. CROSS-DOMAIN "INDEPENDENCE" CLAIM: The correlation matrix shows r>0.98 between several
   domain pairs, explicitly noting "same aetheric circuit." This contradicts the
   independence claim needed for the Fisher combination to produce 9.2σ.

5. σ INFLATION: Page claims 9.2σ in the static HTML audit box but status_history.json
   shows 14.26σ dynamically. Two different numbers for the same statistic.

6. PAGE TITLE CLAIMS "20-domain" BUT: Several domains are unscored/pending. The 100%
   score is computed over scored domains only, excluding failures-to-measure.

7. FALSIFICATION THRESHOLDS: Many are set extremely wide (e.g., Schumann >0.3 Hz drift
   for >1 hour — this has never happened in recorded history). The falsification criteria
   are designed to be impossible to trigger.
