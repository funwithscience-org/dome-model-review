HTTP-Parsing as Exploitation Predictor

HTTP-Parsing vs Non-HTTP Exploitation Rate by CVSS

Exploitation Lift: HTTP-Parsing / Non-HTTP

Critical insight: Products that parse HTTP/untrusted network input are exploited at 3–6× the rate of non-HTTP products at every CVSS severity level. An HTTP-parsing HIGH (7.0–7.9) CVE has a higher exploitation rate than a non-HTTP CRITICAL (9.0–9.9). This suggests "does it parse untrusted input?" is a better triage signal than CVSS score alone.

Why Most Criticals Don't Get Exploited

🔧 Firmware/Hardware

3,739 NVD entries, only 5 KEV (0.13%). Chipset BIOS, printer firmware — high barriers to weaponization, airgapped, update friction.

🐧 Linux Kernel Local

Many criticals require local access. Attackers prefer remote-first vectors — local privesc is a second stage, not a first.

📱 Consumer IoT

Smart home, IP cameras, consumer routers — high CVSS but low strategic value for nation-state/criminal targeting.

🌏 Non-Western Ecosystems

Products popular in specific regional markets (e.g., Huawei, Tenda) get NVD criticals but limited Western exploitation tracking.

💾 Memory Corruption Complexity

Many memory corruption CVEs require precise heap grooming or specific versions — high skill barrier despite high CVSS.

📦 One-CVE-Wonder Products

Niche products with a single critical CVE and minimal deployment. Not worth exploit development for attackers.

Reverse Proxy ≠ Protection

A common assumption: "our apps sit behind Apache/nginx reverse proxy, so library vulns are mitigated."

Reality: Spring4Shell and Log4Shell payloads pass through mod_proxy untouched. The reverse proxy forwards the malicious HTTP headers and request bodies without inspection. WAF rules can help, but only if signatures exist — zero-days bypass WAFs by definition.

The reverse proxy protects against network-layer attacks, not application-layer exploits targeting libraries behind it.

Recommended Patch SLA Framework

⚡ 72 Hours

Any CVE entering CISA KEV. Active exploitation confirmed. No exceptions.

🔥 7 Days

HTTP-parsing Critical/High not yet in KEV. Products: web servers, VPN/firewalls, email servers, app frameworks parsing untrusted input.

📋 30 Days

All other Critical/High CVEs. Standard patch cycle. Deprioritize firmware, consumer IoT, niche libraries unless Internet-facing.
